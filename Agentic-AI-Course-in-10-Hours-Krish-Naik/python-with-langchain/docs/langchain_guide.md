## 1. Prerequisites & Environment Setup

To run the examples, ensure you have installed the required dependencies.

### Installation
Run the following command to install the required LangChain packages and utilities:

```bash
pip install langchain langchain_community langchain-openai langchain-groq langchain-google-genai python-dotenv
```

### Environment Variables
Configure your API keys in a `.env` file at the root of your project:

```env
OPENAI_API_KEY=your_openai_api_key_here
GOOGLE_API_KEY=your_google_api_key_here
GROQ_API_KEY=your_groq_api_key_here
```

In your Python code, load these variables before initializing any model or agent:

```python
import os
from dotenv import load_dotenv

# Load variables from the .env file
load_dotenv()

# Set environment variables for the SDKs to pick up
os.environ["OPENAI_API_KEY"] = os.getenv("OPENAI_API_KEY")
os.environ["GOOGLE_API_KEY"] = os.getenv("GOOGLE_API_KEY")
os.environ["GROQ_API_KEY"] = os.getenv("GROQ_API_KEY")
```

## 2. Model Integration

LangChain supports two main approaches to initialize chat models: **Unified Initialization** (highly flexible and standard) and **Provider-Specific Initialization** (direct import of provider wrapper classes).

### Unified Initialization (Recommended)
The `init_chat_model` utility function allows you to initialize chat models dynamically across multiple providers using standard model name strings. This keeps your code clean and decoupled from specific imports.

```python
from langchain.chat_models import init_chat_model

# 1. OpenAI Integration (e.g. gpt-4o or gpt-4)
openai_model = init_chat_model("gpt-4o")
response = openai_model.invoke("Hello, how are you?")
print("OpenAI Response:", response.content)

# 2. Google Gemini Integration
gemini_model = init_chat_model("google_genai:gemini-2.5-flash-lite")
response = gemini_model.invoke("Why do parrots talk?")
print("Gemini Response:", response.content)

# 3. Groq Integration (e.g., using Qwen)
groq_model = init_chat_model("groq:qwen/qwen3-32b")
response = groq_model.invoke("Why do parrots talk?")
print("Groq Response:", response.content)
```

> [!TIP]
> The `init_chat_model` function uses prefixes like `google_genai:` and `groq:` to map model strings to their respective integrations.

---

### Provider-Specific Integration

If you prefer to import and configure provider-specific classes directly, use the corresponding packages.

#### A. OpenAI (`langchain-openai`)
```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o")
response = model.invoke("Hello, how are you?")
print(response.content)
```

#### B. Google Gemini (`langchain-google-genai`)
```python
from langchain_google_genai import ChatGoogleGenerativeAI

model = ChatGoogleGenerativeAI(model="gemini-2.5-flash-lite")
response = model.invoke("Why do parrots talk?")
print(response.content)
```

#### C. Groq (`langchain-groq`)
```python
from langchain_groq import ChatGroq

model = ChatGroq(model="qwen/qwen3-32b")
response = model.invoke("Why do parrots talk?")
print(response.content)
```

## 3. Messages & Conversation History

Messages are the fundamental unit of context in chat models. They carry content, role information, and metadata.

### Message Types
LangChain provides standard message types representing different roles:

| Message Class | Role / Purpose |
| :--- | :--- |
| `SystemMessage` | Tells the model how to behave and provides system-level context or guardrails. |
| `HumanMessage` | Represents inputs and interactions from the user. |
| `AIMessage` | Represents responses generated by the model (text, tool calls, metadata). |
| `ToolMessage` | Represents the output of a tool execution, passed back to the model. |

### Managing Chat History
You can construct conversations by building a list of message objects and passing them to the model:

```python
from langchain.messages import SystemMessage, HumanMessage, AIMessage
from langchain.chat_models import init_chat_model

model = init_chat_model("groq:qwen/qwen3-32b")

# Build conversation history
messages = [
    SystemMessage("You are a helpful coding assistant."),
    HumanMessage("How do I create a REST API?"),
    AIMessage("You can use Flask or FastAPI. Here is a simple Flask example..."),
    HumanMessage("Great! Can you show it with FastAPI instead?")
]

response = model.invoke(messages)
print(response.content)
```

## 4. Custom Tools & Binding

Tools are interfaces that models use to interact with external environments. They contain both execution logic and a JSON schema description.

### Defining Tools
Define custom tools easily using the `@tool` decorator. 

> [!IMPORTANT]
> The docstring is parsed by LangChain to create the tool's description. The LLM uses this description to decide when to call the tool and what arguments to supply.

```python
from langchain.tools import tool

@tool
def get_weather(location: str) -> str:
    """Get the current weather at a location.

    Args:
        location: The city and state/country.
    """
    return f"It's sunny in {location}"
```

### Binding Tools to Models
To make a model aware of tools, bind them using `bind_tools`. This tells the model that these tools are available to be called.

```python
model = init_chat_model("groq:qwen/qwen3-32b")

# Bind tool to the model
model_with_tools = model.bind_tools([get_weather])
```

### Inspecting Tool Calls
When a tool-bound model is invoked with a query that requires tool use, it generates an `AIMessage` containing `tool_calls`:

```python
response = model_with_tools.invoke("What's the weather like in Boston?")

# Check for tool calls
if response.tool_calls:
    for tool_call in response.tool_calls:
        print(f"Tool Name: {tool_call['name']}")
        print(f"Arguments: {tool_call['args']}")
        print(f"Call ID:   {tool_call['id']}")
```

## 5. Manual Tool Execution Loop

To execute the tool calls requested by the model and supply results back to the model, you can run a manual loop:

```python
from langchain.messages import AIMessage, ToolMessage

# Step 1: User asks query, model requests tool execution
messages = [HumanMessage(content="What's the weather in Boston?")]
ai_msg = model_with_tools.invoke(messages)
messages.append(ai_msg)

# Step 2: Execute the tool based on model requests
for tool_call in ai_msg.tool_calls:
    if tool_call["name"] == "get_weather":
        # Call the tool function with the arguments supplied by the model
        tool_output = get_weather.invoke(tool_call["args"])
        
        # Create a ToolMessage with the output and matching call ID
        tool_msg = ToolMessage(
            content=str(tool_output),
            tool_call_id=tool_call["id"]
        )
        messages.append(tool_msg)

# Step 3: Pass tool outputs back to the model for the final response
final_response = model_with_tools.invoke(messages)
print("Final Response:", final_response.content)
```

## 6. Building and Invoking Agents

For more complex applications where you want the execution loop (Step 5) to run automatically under the hood, use LangChain's built-in Agent utilities.

### Creating an Agent
```python
from langchain.agents import create_agent

# Initialize the model and define the tool list
model = init_chat_model("gpt-4o")
tools = [get_weather]

# Create the agent executor
agent = create_agent(
    model=model,
    tools=tools,
    system_prompt="You are a helpful weather assistant."
)
```

### Invoking the Agent
```python
response = agent.invoke({
    "messages": [
        {"role": "user", "content": "What is the weather in Paris?"}
    ]
})

print("Agent Response:", response)
```
